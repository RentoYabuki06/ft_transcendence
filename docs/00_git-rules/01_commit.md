## コミットの切り方

```bash
git commit -m "<prefix>(<issue-no>): <description>"
```

### prefix とは

どのような操作(編集)を行ったかを示すもの

|番号|名称|説明|
|:-:|:--|:--|
|0|Add|ファイルや機能、説明の追加|
|1|Delete|ファイルや機能、説明の削除|
|2|Fix|ファイルや機能、説明の修正|
|3|Modify|ファイルや機能、説明の変更|
|4|Test|機能や動作のテスト|

<br>

### issue-no とは

ISSUEの番号

<br>

### description とは

コミット内容の説明

- 一行目は大雑把に(必須)
- 二行目以降に細かく(省略可)
